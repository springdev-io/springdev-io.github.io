Spring Session for Apache Geode/Pivotal GemFire 2.0.5.RELEASE 和 2.1.0.M1 同时发布

release 
rain
2018-08-30
评论

Spring Data 团队的John Blum在开发者博客上宣布了Spring Session for Apache Geode & Pivotal GemFire（SSDG）2.0.5.RELEASE（Apple）和 2.1.0.M1（Bean）同时发布！

SSDG 2.0.5.RELEASE 基于 Spring Session 2.0.5.RELEASE，Spring Data Kay-SR9 和 Spring Framework 5.0.8.RELEASE，可从Maven Central获取组件包。

SSDG 2.1.0.M1 基于 Spring Session 2.1.0.M2，Spring Data Lovelace-RC2 和 Spring Framework 5.1.0.RC2，可从Spring lib-milestone获取组件包。

这两个版本都提供了配置Spring Session的新方法，它用来管理Spring Boot Web应用程序（HTTP）会话状态，无论是使用Apache Geode或Pivotal GemFire，还是Pivotal Cloud Cache（PCC）都可以做到。

目前，你如果使用Spring Session来管理 (HTTP) 会话状态，可以选择其中的Apache Geode 或者 Pivotal GemFire，需要在应用的类加载路径中添加对应的库org.springframework.session:spring-session-data-geode 或者是 org.springframework.session:spring-session-data-gemfire。此外，还需要在其中一个Configuration中添加注解@EnableGemFireHttpSession，就像这样：

@SpringBootApplication
@EnableGemFireHttpSession(maxInactiveIntervalSeconds=600)
public class MyWebApplication {

  public static void main(String[] args) {
    SpringApplication.run(MyWebApplication.class, args);
  }
...
}

如果需要修改默认属性，可以在application.properties中添加下列配置：

#application.properties

spring.session.data.gemfire.session.expiration.max-inactive-interval-seconds=600
...

更多配置说明，请阅读参考文档
