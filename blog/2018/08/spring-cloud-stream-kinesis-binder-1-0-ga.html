Spring Cloud Stream Kinesis Binder 1.0 正式发布，这也意味着在生产级Spring应用中，您将可以很轻松的集成AWS Kinesis，并很容易的构建一个消息驱动的微服务应用程序。本文将带您详细了解Amazon Kinesis以及如何在项目中使用。

介绍

Amazon Web Services（AWS）

亚马逊网络服务（Amazon Web Services：AWS）于2006年推出，通过其云计算平台为企业提供关键的基础设施服务。使用云计算业务可以采用新的业务模式，从而无需计划和投资采购自己的IT基础架构。他们可以使用云服务提供商提供的基础架构和服务，并在使用服务时付费。访问http://aws.amazon.com/products/，了解亚马逊作为其云计算服务的一部分提供的各种产品的更多详细信息。

Amazon Kinesis

Amazon Kinesis 可让您轻松收集、处理和分析实时流数据，以便您及时获得见解并对新信息快速做出响应。Amazon Kinesis 提供多种核心功能，可以经济高效地处理任意规模的流数据，同时具有很高的灵活性，让您可以选择最符合应用程序需求的工具。借助 Amazon Kinesis，您可以获取实时数据 (例如视频、音频、应用程序日志、网站点击流) 以及关于机器学习、分析和其他应用程序的 IoT 遥测数据。借助 Amazon Kinesis，您可以即刻对收到的数据进行处理和分析并做出响应，无需等到收集完全部数据后才开始进行处理。

Spring Cloud Stream

Spring Cloud Stream是一个用于构建消息驱动的微服务应用程序的框架。Spring Cloud Stream构建于Spring Boot之上，用于创建独立的生产级Spring应用程序，并使用Spring Integration提供与消息代理的连接。它提供了来自多个供应商的中间件的固定配置，引入了持久化的发布-订阅语义，消费者分组和分区的概念。

Spring Cloud Stream的核心构建组件有：

  • 目标绑定器（Destination Binders）：负责提供与外部消息传递系统集成的组件。

  • 目标绑定（Destination Bindings）：外部消息传递系统和应用程序之间的桥接消息的生产者和消费者（由目标绑定器创建）。

  • 消息（Message）：生产者和消费者用于与目标绑定器（以及通过外部消息传递系统的其他应用程序）通信的规范数据结构。

目前，Spring Cloud Stream支持RabbitMQ和Apache Kafka绑定器，也将包含更多绑定器实现（例如：Google PubSub和AWS Kinesis）。

Spring Cloud Stream Binder for AWS Kinesis

Spring Cloud Stream Binder for AWS Kinesis提供了Spring Cloud Stream的绑定实现。此实现在其基础上使用Spring Integration AWS Kinesis通道适配器。下图解释了Kinesis Binder实现如何将每个配置的目标映射到AWS Kinesis Streams：

与Apache Kafka不同，AWS Kinesis不为消费者分组提供开箱即用的支持。此功能的支持是作为KinesisMessageDrivenChannelAdapter中分片检查点的MetadataStore密钥的一部分来实现的 - [CONSUMER_GROUP]：[STREAM]：[SHARD_ID]。此外，LockRegistry用于确保对每个分片的独占访问。这样，同一个消费者分组中只有一个通道适配器将消费来自其配置的流中的单个分片的消息。

AWS Kinesis中的分区逻辑类似于Apache Kafka支持，但逻辑略有不同。生产者端的partitionKey确定将数据记录分配给流中的哪个分片。分区键是Unicode字符串，每个键的最大长度限制为256个字符。AWS Kinesis使用分区键作为哈希函数的输入，该哈希函数将分区键和关联数据映射到特定分片。具体来说，MD5哈希函数用于将分区键映射到128位整数值，并将关联的数据记录映射到分片。作为此散列机制的结果，具有相同分区键的所有数据记录映射到流中的相同分片。但同时我们无法选择明确发送的目标分片。虽然手动计算哈希值（并分别对producer使用explicitHashKeyExpression），但我们可以通过包含在HashKeyRange中来跟踪目标分片。

默认情况下，分区键是消息载体中Object.hash()的结果。

在AWS Kinesis Binder的情况下，Spring Cloud Stream分区处理逻辑被排除，因为它不再使用，并且提供的producer.partitionKeyExpression直接传播到KinesisMessageHandler。

在消费者方面，instanceCount和instanceIndex用于在组中的消费者之间均匀分配分片。

配置参数

有关特定于Kinesis 绑定器和绑定通道的设置，在项目说明文档中有详细的介绍。

如何使用

在Maven中添加依赖项

如果使用Maven，可以在项目POM的依赖中添加如下，这些组件已经发布到Spring Release和Maven Central了。

<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-stream-binder-kinesis</artifactId>
    <version>1.0.0.RELEASE</version>
</dependency>

spring-cloud-stream-binder-kinesis本身依赖了spring-integration-aws组件，使用其来实现一些集成。如果引入前者依赖项，那么后者可以不用再引入。

关于Spring Integration for AWS 2.0，会有详细介绍说明。
