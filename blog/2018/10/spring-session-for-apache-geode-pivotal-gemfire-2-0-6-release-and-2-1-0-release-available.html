Spring Session for Apache Geode/Pivotal GemFire 2.0.6.RELEASE和2.1.0.RELEASE 同时发布

release 
rain
2018-10-26
评论

Spring Data 团队的John Blum在开发者博客上宣布了Spring Session for Apache Geode/Pivotal GemFire（SSDG）2.0.6.RELEASE和2.1.0.RELEASE同时发布！

SSDG 2.0.6.RELEASE 基于 Spring Framework 5.0.10.RELEASE、Spring Data Kay-SR11 和 Spring Session core 2.0.7.RELEASE等核心依赖升级。

SSDG 2.1.0.RELEASE 基于 Spring Framework 5.1.1.RELEASE、Spring Data Lovelace-SR1 和 Spring Session 2.1.0.RELEASE，另外还包括对自定义会话过期策略和规则的支持。

自定义会话过期配置

Spring Session for Apache Geode & Pivotal GemFire现在提供了一个新的Strategy接口：SessionExpirationPolicy，允许用户根据会话到期的方式和时间自定义规则。

接口大致定义为：

SessionExpirationPolicy
import java.time.Duration;

@FunctionalInterface
interface SessionExpirationPolicy {

  Optional<Duration> determineExpirationTimeout(Session session);

  default ExpirationAction getExpirationAction() {
    return ExpirationAction.INVALIDATE;
  }
}

用户只需要实现determineExpirationTimeout方法，该方法返回一个Optional java.time.Duration，表示Session到期之前的时间长度。

Duration是可选的，因此，在某些情况下，如果您的自定义策略无法确定特定会话的到期超时时间，则返回Optional.empty()将表示使用Apache Geode或Pivotal GemFire的默认为区域管理应用程序会话状态配置的空闲超时时间策略。默认空闲超时时间为30分钟，您可以通过设置@EnableGemFireHttpSession注解的maxInactiveIntervalInSeconds属性来更改。

也可以通过在Spring Boot的application.properties中设置spring.session.data.gemfire.session.expiration.max-inactive-interval-seconds。或者，在Spring容器中声明一个类型为SpringSessionGemFireConfigurer的Bean并且重载getMaxInactiveIntervalInSeconds()方法来实现。

然后，您只需将自定义SessionExpirationPolicy实例声明为Spring容器中的bean，然后通过设置@EnableGemFireHttpSession注解的sessionExpirationPolicyBeanName属性来确保Spring Session for Apache Geode或Pivotal GemFire了解它，如下所示：

@SpringBootApplication
@EnableGemFireHttpSession(
  sessionExpirationPolicyBeanName = "customExpirationPolicy"
)
class MySpringSessionApplication {

  @Bean
  SessionExpirationPolicy customExpirationPolicy(..) {
    return new CustomExpirationPolicy(..);
  }

  ...
}

因为SessionExpirationPolicy实例是一个bean，就像Spring容器中的任何其他托管bean一样，您可以根据需要配置它，注入对其他bean的引用等等。

同样，自定义beanSessionExpirationPolicy的名称可以通过在Spring Boot的application.properties中设置spring.session.data.gemfire.session.expiration.bean-name，或者在Spring的容器中声明一个类型SprinSessionGemFireConfigurer的实例，并且重载getSessionExpirationPolicyBeanName()方法来实现。

更多细节可以在文档中找到。

固定到期时长（Fixed Duration Expiration）

除了允许自定义会话过期策略和规则之外，Spring Session for Apache Geode & Pivotal GemFire还提供了开箱即用的此接口的实现，以支持固定到期超时时间。

这在当会话尚未闲置时，会话应该在一段固定的持续时间（可能是1小时）之后超时的情况下非常有用。

这个需求实际上出现在核心Spring Session 问题＃922 - 在一段时间之后“会话必须超时”。

虽然Spring Session的store providers都无法使用此功能，但可以使用Apache Geode或Pivotal GemFire。

通过以下简单方法来启用：

import org.springframework.session.data.gemfire.expiration.support.FixedTimeoutSessionExpirationPolicy;

@SpringBootApplication
@EnableGemFireHttpSession(
  sessionExpirationPolicyBeanName = "fixedTimeoutExpirationPolicy"
)
class MySpringSessionApplication {

  @Bean
  SessionExpirationPolicy fixedDurationExpirationPolicy() {
    return new FixedTimeoutSessionExpirationPolicyt(Duration.ofMinutes(60L));
  }

  ...
}

当然，Spring Session for Apache Geode & Pivotal GemFire会谨慎地处理Session的空闲超时。因此，只要空闲超时或固定超时已经过去，会话就会到期，这是首先发生的。

同样，可以在文档中找到更多详细信息。
