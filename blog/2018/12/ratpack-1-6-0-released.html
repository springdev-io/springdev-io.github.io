Ratpack 1.6.0 发布

release 
rain
2018-12-20
评论

以下引用自官方网站：

Ratpack 是一组用于构建可伸缩 HTTP 应用程序的 Java 库。 它是一个简洁而强大的基础库，而不是一个包罗万象的框架。

Ratpack 是一组 Java 库，可以促进快速、高效、可演化和经过良好测试的 HTTP 应用程序。 它建立在高性能和高效的 Netty 事件驱动的网络引擎上。

Ratpack 纯粹是一个运行时。没有可安装的软件包，也没有耦合的构建工具（例如 Rails、Play、Grails）。 要构建 Ratpack 应用程序，您可以使用任何 JVM 构建工具。Ratpack 项目通过插件为 Gradle 提供特定支持，但任何其他工具也都可以使用。

Ratpack 提供一系列组件库，以 JAR 的形式发布。ratpack-core 库是唯一必须要求的。 其他如 ratpack-groovy、ratpack-guice、ratpack-jackson、ratpack-test 等都是可选的。

Ratpack 的目标是：

  • 快速，可扩展，高效

  • 允许应用程序在不妥协的情况下进化复杂性

  • 利用非阻塞编程的好处，并降低成本

  • 在集成其他工具和库时要灵活且不受任何影响

  • 允许应用程序轻松、彻底地进行测试

Ratpack 的目标不是：

  • 成为一个完全集成的“全栈”解决方案

  • 在一个框架中提供您可能需要的所有功能

  • 为“业务逻辑”提供架构或框架

Ratpack 是强类型的。除了在 Java（一种强类型语言）中实现之外，它的 API 也拥抱并支持类型。 例如，注册表的概念在 Ratpack 中广泛使用。可以将注册表视为使用类型作为键的映射。

对于在 Groovy 中实现其应用程序的 Ratpack 用户来说，这可能是最感兴趣的。 Ratpack 的 Groovy 适配器使用最新的 Groovy 特性来完全支持静态类型，同时保持惯用的、简洁的 Groovy API。

Ratpack 的核心是一个基于事件（即非阻塞）的 HTTP IO 引擎，以及一个可以轻松构建响应逻辑的 API。 非阻塞强加了与“传统”阻塞 Java API 不同的 API 风格，因为 API 必须是异步的。

Ratpack 旨在高效简化 HTTP 应用程序的这种编程风格。它为构造异步代码提供了支持，并使用一种创新方法将请求处理结构化为自建、异步遍历、函数图（这很少见，因为它听起来感觉很难使用）。

Ratpack 1.6.0 现已发布！ 此版本增加了与其他响应式编程库的集成以及许多改进，使 Ratpack 可以更加无缝地与外部库集成。

新的 ratpack-reactor 模块增加了对 Ratpack 与 Project Reactor 类型集成的支持。

新的 ratpack-rx2 模块增加了对将 Ratpack 与 RxJava 2 版本集成的支持。 这是一个支持 RxJava 2.0 中引入的新类型的新模块。

新的 ratpack-thymeleaf3 模块增加了对 Ratpack 与 Thymeleaf 模板 3.x 版本集成的支持。

新的 ratpack-gson 模块增加了对使用 Google 的 Gson 库解析请求主体和响应内容的支持。

此版本还添加了许多其他新的便捷方法、错误修复和通常很酷的东西。

详细内容参见：1.6.0 发布说明

升级依赖项：

  • Netty 4.1.32.Final

  • Jackson 2.9.8

  • ReactiveStreams 1.0.2

  • Caffeine 2.6.2

  • SnakeYAML 1.23

  • Handlebars 4.0.7

Ratpack 1.7 目前已经在开发之中，2.0 也在规划和重构，将会支持 Java 9+ 的模块化功能，值得期待。

我曾经使用 1.3 版本，写过一个博客系统和小型电商网站，开发速度和体验都非常不错。 目前也有参与提交一些 PR 和问题到项目，值得推荐给 Java 和 Groovy、Spring 开发者，尤其是用过 Groovy。 在国外，也有类似 SmartThings 的创业公司大量使用。 注明：https://www.smartthings.com，一家做智能家用产品的公司，现已被三星公司收购。
