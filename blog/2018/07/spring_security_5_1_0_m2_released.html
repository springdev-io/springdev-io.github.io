Spring Security 5.1.0 M2发布了

release 
rain
2018-07-30
评论

Spring Security的项目负责人Rob Winch在官方博客宣布了5.1.0.M2版本发布的消息，这个版本总共关闭了100多个任务单，其中有不少新特性。

下面我们将一起来看看有哪些亮点。

OAuth2

OAuth2资源服务器（OAuth2 Resource Server）

添加了对OAuth2资源服务器的基本支持。 参见示例：oauth2resourceserver

Authorization Code Flow

用户现在可以使用OAuth 2.0授权代码授予获取访问令牌。参见示例： authcodegrant

支持OAuth2和WebClient

现在内置了对OAuth2和WebClient的支持。具体如下：

  • 添加访问令牌到请求中

  • 访问令牌过期时自动刷新

  • 解析要使用的访问令牌

例如，在Servlet环境中，你可以像这样配置Bean：

@Bean
WebClient webClient(OAuth2AuthorizedClientRepository repository) {
    ServletOAuth2AuthorizedClientExchangeFilterFunction filter =
        new ServletOAuth2AuthorizedClientExchangeFilterFunction(repository);
    return WebClient.builder()
        .filter(new OAuth2AuthorizedClientExchangeFilterFunction())
        .apply(filter.oauth2Configuration())
        .build();
}

现在，你可以通过多种不同方式添加OAuth令牌。 如果需要，可以使用Spring MVC支持解析OAuth2AuthorizedClient。如果授权服务器返回刷新令牌，而访问令牌也即将过期时，Spring Security则将透明地更新访问令牌并提交更新后的访问令牌。

@GetMapping("/users")
Mono<String> users(@RegisteredOAuth2AuthorizedClient("client-id")
        OAuth2AuthorizedClient authorizedClient) {
    return this.webClient.get()
        .uri("https://api.example.com/user")
        .attributes(oauth2AuthorizedClient(authorizedClient))
        .retrieve()
        .bodyToMono(String.class);
}

你还可以通过“WebClient”解析访问令牌。 例如：

Mono<String> users() {
    return this.webClient.get()
        .uri("https://api.example.com/user")
        .attributes(clientRegistrationId("client-id"))
        .retrieve()
        .bodyToMono(String.class);
}

如果使用OAuth2登录或OIDC进行身份验证，则可以获取默认访问令牌而无需用户交互。

Mono<String> users() {
    // if Authenticated with OIDC
    // OAuth2 Log In use the access token associated to log in
    return this.webClient.get()
        .uri("https://api.example.com/user")
        .retrieve()
        .bodyToMono(String.class);
}

WebFlux OAuth2登录支持OIDC

WebFlux应用程序现在可以使用OAuth2和OIDC进行身份验证。 参见示例： oauth2login-webflux

能够从OIDC Discovery创建ClientRegistration

之前，用户希望与OIDC提供程序集成需要配置所有端点。例如，在Spring Boot中需要这样做：

spring:
  security:
    oauth2:
      client:
        registration:
          okta:
            client-id: okta-client-id
            client-secret: okta-client-secret
        provider:
          okta:
            authorization-uri: https://foo.oktapreview.com/oauth2/v1/authorize
            token-uri: https://foo.oktapreview.com/oauth2/v1/token
            user-info-uri: https://foo.oktapreview.com/oauth2/v1/userinfo
            user-name-attribute: sub
            jwk-set-uri: https://foo.oktapreview.com/oauth2/v1/keys

现在，用户可以简单的配置发行者uri。例如:

spring:
  security:
    oauth2:
      client:
        registration:
          okta:
            client-id: okta-client-id
            client-secret: okta-client-secret
        provider:
          okta:
            issuer-uri: https://foo.oktapreview.com/oauth2/default/

支持自定义授权请求

添加了OAuth2AuthorizationRequestResolver用于支持自定义授权请求。 例如，如果用户想要从请求中添加/删除范围，他们现在可以轻松地执行此操作。

单个ClientRegistration将会重定向到登录

使用单个提供程序配置OAuth2/OIDC登录时，不是仅显示一个提供程序，而是更改默认值以立即重定向到提供程序。

配置改进

现代化默认登录页面

默认登录页面已经采用html5编写，看起来更具视觉吸引力。

默认注销页面(Default Log Out Page)

由于添加了CSRF注销保护，因此默认应用程序无法注销。 现在，如果正在使用默认登录页面（即没有配置登录页面），那么还有一个默认的注销页面，它显示了一个注销表单。

简化RequestCache配置

用户现在可以配置默认的RequestCache，将其暴露为@ Bean。 已经更新了HttpSessionRequestCache，不再要求该值为DefaultRequestCache。

强化你的应用程序(Hardening Your Application)

跨站点跟踪和HTTP动词篡改保护（Cross Site Tracing & HTTP Verb Tampering Protection）

Spring Security 5.1 添加了 跨站点跟踪 和 HTTP动词篡改保护.

密码编码升级（Password Encoding Upgrades）

用户可以实现UserDetailsPasswordService 并暴露为 @Bean，当认证成功后Spring Security的DaoAuthenticationProvider将会：

  • 检查密码存储机制是否需要使用新的PasswordEncoder.upgradeEncoding方法来更新。例如， 如果它是采用sha256中编码的，那么默认情况下，Spring Security会建议它升级到BCrypt。

  • 如果需要升级密码编码，它将使用当前的PasswordEncoder

  • UserDetails和新密码将被传递给UserDetailsPasswordService，因此可以使用升级的密码编码保存。

依赖更新

我们已经更新了框架的依赖关系，以确保框架的传递依赖关系是最新的。
