Spring Boot for Apache Geode和Pivotal GemFire 1.0.0.M3 发布

release 
rain
2018-09-22
评论

Spring Boot for Apache Geode和Pivotal GemFire 1.0.0.M3（以下简称：SBDG）发布。

在1.0最终GA之前的所有特性已经基本完成了，在最终GA之前还有1个开发里程碑（RC1）。

新特性

SBDG 1.0.0.M3增加了对Apache Geode和Pivotal GemFire（以及Pivotal Cloud Cache，由Pivotal GemFire支持）使用Spring Boot的Actuator功能的支持。

具体来说，这个支持包括Spring Boot HealthIndicators，用于评估以下GemFire/Geode组件的状态和运行状况：

总的来说，这个版本带来了以下改进：

  • Cache

  • DistributedSystem / DistributedMember

  • ResourceManager

  • Regions

  • Indexes

  • DiskStores

  • Continuous Queries

  • CacheServers

  • Pools

  • AsyncEventQueues

  • GatewaySenders

  • GatewayReceivers

除基本状态信息外，GemFire/Geode HealthIndicators还提供基本配置元数据详细信息以及GemFire/Geode组件本身的运行时特征和行为。这包括简单的指标，如缓存区域命中/未命中计数或客户端和服务器之间的连接负载，等等。

“必要的”配置元数据详细信息在一个方便的位置提供，因此您无需筛选所有命令行参数、日志文件、属性文件和其他配置文件，只需查找每个GemFire/Geode的配置在应用程序架构中使用的对象。

有关提供的HealthIndicators的更多详细信息，请访问此处。

示例

还有一个未记录的示例，你可以在samples分支上查看。示例Spring Boot应用程序是一个用于温度传感器的简单温度监控服务，并使用Apache Geode CQ功能将温度事件发送到客户端（监视器）。它使用足够的Geode组件来使Actuator健康端点变得有趣。

如何使用

您可以通过Spring libs-milestone repo来获取新包，使用的话需要在Maven中增加以下依赖项：

  • org.springframework.geode:spring-geode-starter （Apache Geode）

  • org.springframework.geode:spring-gemfire-starter （Pivotal GemFire/PCC）

后续计划

通过RC1，我们打算完成以下内容：

  • 更多示例，并附有文档

  • 扩大测试范围，涵盖所有UC方式

  • 评审文档的正确性和简洁性
