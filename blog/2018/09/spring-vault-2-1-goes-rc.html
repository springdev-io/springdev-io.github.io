Spring Vault 2.1 RC1 发布

release 
rain
2018-09-10
评论

Spring Vault提供熟悉的Spring抽象和客户端支持，用于访问、存储和撤销机密。它提供了与Vault交互的低级和高级抽象，使用户免于底层接口问题。

通过HashiCorp的Vault，你可以在所有环境中为应用程序管理外部机密数据。Vault可以管理静态和动态机密，例如应用程序数据，远程应用程序/资源的用户名/密码，并为外部服务提供凭据，如MySQL、PostgreSQL、Apache Cassandra、Consul、AWS等。

Spring Vault 2.1需要JDK 8或更高版本，并且特别支持JDK 11作为下一个长期支持版本。这个版本完成了15张任务单，以及支持Java 9到11所做的一些改进。

其中有一些值得注意的变更有：

  • 支持在VM上使用托管服务实例进行Azure身份验证

  • 支持以服务帐户作为GCP IAM身份验证

此版本与Vault版本从0.5.2到0.11.1都兼容。

第三方依赖升级如下：

  • Spring Framework 5.1 RC3

  • Spring Data Lovelace RC2

  • Reactor Californium RC1

  • netty 4.1.29.Final

  • Google IAM v1-rev259-1.25.0

  • AWS SDK 1.11.404

  • Vault 0.11.1

开发团队计划在10月初发布GA版本，将会修正错误以及对依赖项进行升级。同时也正在努力推出下一个Spring Vault 2.2版本，以便为版本化的Vault键值后端接口提供基于注解的配置。
