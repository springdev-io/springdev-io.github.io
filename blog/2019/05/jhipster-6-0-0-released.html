JHipster v6.0.0 发布

release 
rain
2019-05-02
评论

JHipster 是一个开发平台，用于生成、开发和部署 Spring Boot + Angular/React Web 应用程序和 Spring 微服务。

该框架的目标是为开发者生成一个完整并统一的现代 Web 应用程序或微服务架构：

  • 使用 Spring Boot 在服务器端实现高性能和强大的 Java 技术栈

  • 这是一款时尚、现代、移动优先的前端框架，支持 Angular、React 和 Bootstrap

  • 一个强大的微服务架构，包括 JHipster Registry、Netflix OSS、ELK 和 Docker

  • 使用 Yeoman、Webpack 和 Maven/Gradle 构建应用程序的强大工作流程

这个作为重要的大版本 6.0.0，总共在 520 多个问题单和代码合并请求。

  • 迁移到 Spring Boot 2.1.x

  • 迁移到 Spring Cloud Greenwish.x

  • 升级到 Spring Security 5.1 对 OIDC 提供的支持

  • 支持 JDK 11 (同时保持兼容 JDK 8+)

  • HTML 5 pushstate

  • Kubernetes 相关增强 (Istio, Helm)

  • 升级到 JUnit 5

  • 支持使用 FakerJS 生成实体相关的样例数据

  • 升级到最新 Angular 版本

  • 升级到最新 React 版本

  • 支持懒加载 Angular 实体

  • 提供 Bootswatch 主题选择

  • 删除 CSS 选项

  • 改善与 Sonar 的集成, 提供了外部使用的 sonar-project.properties 文件

  • 支持 Gatling 3，包括一些更好更快的增量构建，以及 BOM 支持

  • 集成测试在 Maven 和 Gradle 的单独阶段设置

  • 升级到 Gradle 5

  • 迁移到 Liquibase 3.6.x

  • 升级到 Elastic to 6.4.x

  • 升级到 Couchbase 6.x

  • 升级到 Infinispan 9.4.x

  • 升级到 Cassandra 4.x

  • 升级到 Hazelcast 3.11.x

  • 输出到控制台的日志采用 JSON 格式

  • 默认采用 Jar 形式打包，同时仍然支持输出 War 包

  • 使用 Prettier 来格式化 YAML 文件

  • 使用 Prettier transform 来美化所有的 sub-generators

与此同时，一些特性在这个版本被删除：

  • 删除已废弃的 ‘rancher-compose’ sub-generator

  • 删除 Chocolatey 和 Homebrew 安装方式，由于它们并没有给用户提供更多好处

  • 暂时废弃 JHipster Devbox：社区正在寻找这部分功能的维护者，如果您感兴趣可以与之联系！
