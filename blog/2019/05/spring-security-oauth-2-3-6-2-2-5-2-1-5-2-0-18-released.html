Spring Security OAuth 2.3.6, 2.2.5, 2.1.5, 2.0.18 发布

release 
rain
2019-05-30
评论

Spring Security OAuth 2.3.6, 2.2.5, 2.1.5 和 2.0.18 一并发布，主要修复安全问题： CVE-2019-11269: Open Redirector in spring-security-oauth2，建议受此影响的用户尽快升级。

Spring Security OAuth 版本在 2.3 之前的 2.3.6，2.2 之前的 2.2.5，2.1 之前的 2.1.5 和 2.0 之前的 2.0.18 ，以及较旧的不受支持的版本可能容易受到可能泄漏的开放重定向器攻击授权码。恶意用户或攻击者可以使用授权代码授予类型向授权端点发出请求，并通过 redirect_uri 参数指定操纵的重定向 URI。这可能导致授权服务器使用泄漏的授权代码将资源所有者用户代理重定向到受攻击者控制的 URI。

满足以下所有要求的应用程序将导致引发此漏洞：

  • 作为 Authorization Server (例如 @EnableAuthorizationServer)

  • 在 AuthorizationEndpoint 中使用了默认的 DefaultRedirectResolver

下列情况将不会导致漏洞公开：

  • 作为 Authorization Server，但是使用了其他 RedirectResolver 实现，而不是默认的 DefaultRedirectResolver

  • 只是作为 Resource Server (例如 @EnableResourceServer)

  • 只是作为 Client 使用 (例如 @EnableOAuthClient)

详细的安全更新说明在这里：

  • https://pivotal.io/security/cve-2019-11269

有关每个版本中包含的其他更改，请参阅：

  • 2.3.6

  • 2.2.5

  • 2.1.5

  • 2.0.18
