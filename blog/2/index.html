Spring Data Moore M2 发布

Spring Data 开发团队的 Christoph Strobl 目前在博客上宣布了 Moore 的第二个里程碑版本已发布。

值得注意的变化包括有：

  • Spring Data MongoDB、Cassandra 和 Redis 中支持 Kotlin 协程（Coroutines）

  • Querydsl 支持响应式存储库

  • 支持在 Spring Data MongoDB & Cassandra 派生的存储库查询方法 between 使用 Range 类型

  • Neo4j 存储库方法支持 exists 投影

  • Spring Data Elasticsearch 提供了 响应式存储库

  • Pivotal Gemfire 和 Apache Geode 版本升级

  • 升级到 Spring HATEOAS 1.0 M1

  • JDBC module 的其他一些新特性，诸如跳过 是否新数据 检查，直接 insert 和 update

rain

评论
Spring Vault 2.1.2 发布，这些版本带来了错误修复和依赖版本更新。

Spring Vault提供熟悉的Spring抽象和客户端支持，用于访问、存储和撤销机密。它提供了与Vault交互的低级和高级抽象，使用户免于底层接口问题。

通过HashiCorp的Vault，你可以在所有环境中为应用程序管理外部机密数据。Vault可以管理静态和动态机密，例如应用程序数据，远程应用程序/资源的用户名/密码，并为外部服务提供凭据，如MySQL、PostgreSQL、Apache Cassandra、Consul、AWS等。

Spring Vault 2.1.2 已发布，可以在 Maven Central 下载。

这些版本带来了错误修复和依赖版本更新。Spring Cloud Vault 2.1.2 将在接下来的几天内通过 Spring Cloud Greenwich SR1 发布来使用 Spring Vault 2.1.2。

有关更改的完整列表，请参阅更改日志：

Spring Cloud Finchley.SR3 发布

Spring Cloud Finchley SR3（Service Release）已经发布，可以在 Maven Central 上获取到。您可以查看Finchley 发布说明以获取更多信息。

模块更新说明

具体而言，下面是每个更新模块的详细介绍。

Spring Cloud Config

  • Spring Cloud Config 服务器现在在检索数据之前遵循请求中的 if-modified-since 标头，并在响应中返回 last-modified 标头

  • 缺陷修复

Spring Cloud Stream

  • Elmhurst.SR2 发布说明

Spring Cloud Sleuth

  • 缺陷修复

Spring Cloud Zookeeper

  • 缺陷修复

Spring Cloud AWS

  • 缺陷修复

Spring Cloud Task

  • 依赖升级

  • 缺陷修复

Spring Cloud Commons

  • 缺陷修复

Spring Cloud Openfeign

  • 缺陷修复

Spring Cloud Vault

  • 依赖升级

  • 文档更新

  • 缺陷修复

Spring Cloud Bus

  • 缺陷修复

Spring Cloud Contract

  • 缺陷修复

Spring Cloud Consul

  • spring.cloud.discovery.enabled=false 现在等同于 spring.cloud.consul.discovery.enabled=false

  • 缺陷修复

Spring Cloud Gateway

  • 当 Ribbon 不在类路径上时，提供更好的错误处理
