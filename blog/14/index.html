Spring 开发者们每周必读的《Spring一周回顾》，本期为您介绍Micronaut框架，构建微服务应用新选择。

大家好，Spring 粉丝们，感谢一如既往的等待，又到了每周必读的《Spring一周回顾》时间。

在Java领域，开源的框架一直都非常多，从WebWork、Struts、Tapestry、Grails、Dropwizard、Ratpack、Spark，包括Spring Boot，这些都是非常优秀的开源框架。Micronaut被大多数Spring&Grails的开发者寄予厚望，今年5月23日来自OCI的Graeme Rocher宣布了Micronaut正式开源之后，开发团队一直非常努力投入开发之中，在接连发布了M1、M2、M3、M4以及RC1、RC2等版本之后，终于离最终正式版越来越近了，官方计划在10月16号发布第三个候选版本，不出意外将在10月23日迎来最终的GA版发布。欢迎关注本网站，后续会继续报道最新资讯和推出相应的学习教程。

本文的作者是Graeme Rocher，他是Grails和Micronaut框架的项目负责人，主导Grails开发10多年，有着丰富的Grails和Spring架构经验。由于Grails和Micronaut的开发团队和社区与Spring项目之间都很有渊源，而我也是其中最早的用户和参与者，所以本文将Micronaut介绍给Spring技术社区，后续也会发布更多相关的技术文章，欢迎大家积极参与讨论和分享。

以下内容来自OCI官方博客，由Graeme Rocher发表：

继上周的Micronaut 1.0.0 RC1发布之后，我们很高兴地宣布推出Micronaut 1.0.0 RC2，其中包括社区反馈后的改进（感谢所有报告问题的人！）。

第二个RC提供了另一个机会来测试Micronaut并提供反馈，因为我们建立了GA版本计划，将于10月23日发布 - 正好赶上旧金山的Oracle Code One 2018，我将在大会上展示Micronaut！

此外，如果您在欧洲，旧金山不远，请考虑参加Voxxed Days Paris，我将于10月30日在那里展示Micronaut。

在我之前的文章中，我详细阐述了我们使用Micronaut进行的一些设计选择，尽可能避免使用反射，以及这对开发人员和应用程序的性能有何益处。

在这篇文章中，我将详细阐述传统Java框架面临的挑战以及Micronaut如何解决这些挑战。

Spring Cloud Open Service Broker 3.0.0.M1 已发布

Spring Cloud Open Service Broker是一个用于构建实现Open Service Broker API的Spring Boot应用程序的框架。

Open Service Broker API项目允许开发人员为云原生平台（如Cloud Foundry，Kubernetes和OpenShift）中运行的应用程序提供服务。Spring Cloud Open Service Broker提供了一个基于Spring Boot的框架，使你能够在支持Open Service Broker API的平台上为你自己的托管服务快速创建服务代理。
