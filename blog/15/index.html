Spring Boot for Apache Geode 和 Pivotal GemFire 1.0.0.M2 发布

release 
rain
2018-08-31
评论
Spring Boot for Apache Geode 和 Pivotal GemFire 1.0.0.M2 发布，可以从里程碑库中获取。

Spring Boot for Apache Geode 和 Pivotal GemFire 1.0.0.M2 发布。

总的来说，这个版本带来了以下改进：

  • 支持Spring Session的自动配置，当使用Apache Geode 或者 Pivotal GemFire / Pivotal Cloud Cache (PCC)来管理Spring Boot应用程序的 (HTTP) 会话状态

  • 使用Spring Boot自动配置的Spring缓存抽象时，可识别spring.cache.type

  • 对文档的补充和改进

  • 升级Spring Framework 5.0.8.RELEASE，Spring Boot 2.0.4.RELEASE 和 Spring Data Kay-SR9

此版本基于最近推出的Spring Session for Apache Geode/Pivotal GemFire (SSDG) 2.0.5.RELEASE构建。

SSDG的一个关键附加功能是使用Properties和一个新的SpringSessionGemFireConfigurer回调接口，来启用Apache Geode / Pivotal GemFire / PCC 动态配置Spring Session。这样，即使在Spring Boot的自动配置环境中，用户也可以自定义配置以满足其应用程序要求。

Spring Batch 4.1.0.M3 发布

release 
rain
2018-08-31
评论
Spring Batch 4.1.0.M3 发布，可以从里程碑库中获取。

Spring Batch 4.1.0.M3 发布，本次发布带来了如下特性：

支持 JSR-305

这个里程碑的主题是添加对JSR-305注解的支持。利用Spring Framework的Null-safety注解，并在Spring Batch的所有公共API中添加它们。

这些注解不仅在使用Spring Batch API时强制执行null安全性，而且还可以由IDE用于提供与可空性相关的有用信息。例如，如果用户想要实现ItemReader接口，那么任何支持JSR-305注解的IDE都将生成如下内容：

public class MyItemReader implements ItemReader<String> {

    @Nullable
    public String read() throws Exception {
        return null;
    }

}

在read方法添加@Nullable 注解，明确表示此方法可能返回null。这正如在Javadoc中说明的那样，即当数据源处理完成时，read方法应返回null。
