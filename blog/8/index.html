Spring Session for Apache Geode/Pivotal GemFire 2.0.8.RELEASE 发布

Spring Data 团队的 John Blum在开发者博客上宣布了 Spring Session for Apache Geode/Pivotal GemFire（SSDG）2.0.8.RELEASE 发布！

SSDG 2.0.8.RELEASE 基于 Spring Framework 5.0.12.RELEASE、Spring Data Kay-SR13 和 Spring Session core 2.0.9.RELEASE 以及 Apache Geode 1.2.1 和 Pivotal GemFire 9.1.1 等核心依赖升级，SSDG 2.0.8.RELEASE 兼容基于 Spring Boot 2.0.x 的应用程序。

此版本从 2.1.x 版本分支中添加了许多增强功能以及一些小修复。亮点包括：

  1. Prevent saving non-dirty Sessions (Issue-12). This primarily protects against Apache Geode/Pivotal GemFire bug GEODE-6032.

  2. Standardize all logging on SLF4J (Issue-18).

  3. Fixed bug involving incorrect synchronization lock on Session attributes passed down from the Session object on creation.

  4. Fixed bug involving Data Serialization to properly identify when a user uses a custom DataSerializer to de/serialize the Session.

Spring 开发者们每周必读的《Spring一周回顾》。

大家好，Spring 粉丝们，感谢一如既往的等待，又到了每周必读的《Spring一周回顾》时间。

本周三，Spring Cloud 终于迎来了 Greenwich 版本的 正式发布。这样，所有相关的项目都已经兼容 Java 11，此外，Greenwich 还包括了两个新项目： Spring Cloud GCP 和 Spring Cloud Kubernetes。 前者为 Google Cloud Platform（GCP）的服务提供了集成支持，后者提供了 Kubernetes 的集成。在之前的版本发布说明中也多次提到，Spring Cloud Netflix 项目已经进入维护模式，意味着 Spring Cloud 团队将不再向模块添加新功能。但是还将持续维护至少一年的时间，除了会修复重大程序错误和安全问题，还有社区提供的小的 PR。 当然，Spring Cloud 团队也推荐了一些可替换项目，作为迁移的备选方案。比如，Spring Cloud Gateway 可替换 Zuul，Spring Cloud Loadbalancer 可替换 Ribbon，Resilience4j 可替换 Hystrix 等。
